A Unity endless runner with dynamic 3D paths and multiple turn types.

Project Overview

This endless runner game delivers classic features like regenerating levels and obstacles, but it's elevated by all my recent learnings. OOP makes adding new levels, obstacles, and power-ups easy, while Scriptable Objects separate data from logic. I used design patterns like Singleton and Observer pattern.

The environment has a curved shader for a unique feel, and animations are optimized with DOTween. The player movement uses a Blend Tree for smooth transitions. Mobile touch controls let players swipe intuitively. Plus, it's fully optimized with SRP, batching, GPU instancing, and a production-ready object pool. In short, it's production-ready and polished!

Key Features

  • Mobile-friendly controls
  • Scriptable Object-driven architecture to separate game data from gameplay logic
  • Event-based system for decoupled communication and gameplay event triggering
  • Singleton managers (GameManager, SoundManager, etc.) for centralized game control
  • High-performance object pooling system for endless environment spawning
  • Mobile-optimized rendering using Universal Render Pipeline (URP)
  • SRP Batcher-friendly materials and shaders to minimize draw calls
  • Custom Shader Graph shaders for stylized visuals and curved world effect
  • Smooth gameplay flow using tween-based animations
  • Clean OOP-based code architecture following scalable design patterns
  • Efficient endless level regeneration with optimized memory management
  • GitHub for version control and collaboration.
